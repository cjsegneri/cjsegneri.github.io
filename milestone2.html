Milestone 2 - Requirements Analysis Phase

Connor Segneri


Introduction

The NCMEC Data Map is a Shiny-R application that uses missing persons and attempted kidnapping data from the National Council of Missing and Exploited Children to display an interactive map with filterable characteristics for police officers to use in order to more effectively investigate and prevent kidnapping attempts.

This milestone covers the requirements analysis phase for the NCMEC Data Map. The system is further fleshed out in this report. The specific details concerning data characteristics, system flow, technical requirements and more are covered below.

System Design

Diagram


Block Explanations

  1. Import data from NCMEC storage
    1. The data that will be used in this application is stored by NCMEC, at this step the data is going to be read into the application.
  2. Display initial map with all data nodes visible
    1. The main part of the application is going to be an interactive map of the US. I this step, the initial map is shown using the nodes that are read in from the NCMEC data.
  3. Display data filtering options
    1. On the left side of the screen the filtering options are going to be displayed. Here the user can select which characteristics he wants the data nodes to be displayed by.
  4. Display download buttons
    1. The download buttons are going to appear on the top left part of the application once it is loaded. The user can click here to download the data that they currently have filtered.
  5. Display updated map based on filtering selections
    1. Once a filtered option is selected, the map is reloaded only showing the data that passes through the user-selected filters.
  6. Generate CSV with filtered data
    1. This happens when the download button is selected, the data is downloaded that is currently filtered.

Technical Requirements

There are some technical requirements that the system must fulfill that the application is going to be running on. These requirements are listed below.

Requirements

  1. Full access to all NCMEC data.
  2. Enough resources to handle the traffic that the shiny application will get.
    1. This will be determined later, once more information is received about how many police departments across the country NCMEC is partnered with.

Hardware/Software

  1. NCMEC has an IT team with a platform in mind to launch the application from, so more details will be added in the future once more information is received from them.

Once all of the requirements listed above are met, then work on the application can begin, and the application will have an acceptable environment to be hosted within.

Screen Definitions and Layouts

Interactive Map

The map is going to be white and easy to read, as this application will be for police officers who will most likely need this data in a hurry. The nodes are going to be displayed as dots on the map. Whenever a node is clicked on the data for that incident will appear. The map is also going to be scalable, so the user will be able to zoom in and out.


Filtering Options

The filtering options are going to be located on the right side of the application. The filtering options for each data set is going to have its own tab, and the options within these will also be grouped into tabs.

Security

There are numerous security issues that can take place within many different aspects of the application. Some of these potential security issues are detailed below.

Site Hosting Issues

  1. The application is going to have direct access to the database, but it needs to be set up in a way where the application itself cannot alter any of the data.

Potential Coding Issues

  1. If someone can access the code that runs the R shiny application, then they can change what the application does. This could lead to any number of problems with the app itself.

Unauthorized Access

  1. If criminals access the data stored in this application, then they might be able to predict where police officer presence may be heaviest, and they could plan their criminal activity in such a way as to avoid getting arrested.
  2. Only police officers and employees of NCMEC should have access to this data.
