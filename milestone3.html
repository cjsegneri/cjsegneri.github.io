Connor Segneri


Introduction

The NCMEC Data Map is a Shiny-R application that uses missing persons and attempted kidnapping data from the National Council of Missing and Exploited Children to display an interactive map with filterable characteristics for police officers to use in order to more effectively investigate and prevent kidnapping attempts.

This milestone covers the final plan for the NCMEC Data Map. Development for the application begins now. A summary for all the preliminary plans are covered below.

Design Plan Summary

The project that will be completed is called the NCMEC Data Map. The data map is going to be a Shiny application developed in the R programming language. It is going to display a geographical visualization of kidnapping and missing person cases that involve children. The users will be able to filter the data and visually see the changes that occur on the map because of the filters. Finally, the users will be able to download the data that they have filtered. The data will be provided by the National Council for Missing and Exploited Children. They needed a comprehendible way for police officers to access and filter through their data. This application will provide this service.

Overview of Design Concepts

Interactive Map

The map is going to be white and easy to read, as this application will be for police officers who will most likely need this data in a hurry. The nodes are going to be displayed as dots on the map. Whenever a node is clicked on the data for that incident will appear. The map is also going to be scalable, so the user will be able to zoom in and out.


Filtering Options

The filtering options are going to be located on the right side of the application. The filtering options for each data set is going to have its own tab, and the options within these will also be grouped into tabs.

Requirements for System Security

  1. Maintain up to date software.
  2. Take necessary precautions against SQL injections.
  3. Take necessary precautions against cross-site scripting.
  4. Minimalize the information given when showing error messages to users.
  5. Ensure that form validation takes place on both the browser and server side.
  6. Require users to use complex passwords to log in.
  7. Use a complex hashing algorithm to store passwords.
  8. Use the HTTPS protocol to deliver content to users.
