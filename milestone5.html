Milestone 5 - Testing

Connor Segneri


Introduction

The NCMEC Data Map is a Shiny-R application that uses missing persons and attempted kidnapping data from the National Council of Missing and Exploited Children to display an interactive map with filterable characteristics for police officers to use in order to more effectively investigate and prevent kidnapping attempts.

This milestone covers the testing phase for the NCMEC Data Map. The application might have bugs that need to be fixed before deployment.

Component Testing

Description

“Component testing is a method where testing of each component in an application is done separately. Suppose, in an application there are 5 components. Testing of each 5 components separately and efficiently is called as component testing.” -http://istqbexamcertification.com/what-is-component-testing/

During component testing, all objects are tested to ensure they work together as specified by the physical design. If issues are discovered, the project returns to the implementation phase for corrective action.

Unit Testing

Description

“A unit test is the smallest testable part of an application like functions, classes, procedures, interfaces. Unit testing is a method by which individual units of source code are tested to determine if they are fit for use.” -http://istqbexamcertification.com/?s=unit+testing

Test each module individually. Detailed scripts are used in the development and test phases for evaluating the completeness and correctness of the smallest parts of the system and the components created from those parts. Also included are test scripts that are high-level and more focused on processes.

Testing

Data Collection

This app currently imports data from two csv files. If the nodes show up on the map, then the data import correctly.

Results

The data nodes appeared on the application when it was loaded. The data collection procedure works as planned.

Visualized Results
Analytical Engine

The analytical engine is what filters the data based on the characteristics that are entered by the user. To test if this works, I’ve displayed and gone through the data that is being filtered in the back end. I’ve verified that my analytical engine does what it is supposed to do by analyzing the specific data output that it returns based on various filtering options.

Output

Testing the output is simply testing to see if the appropriate data displays on the map based on the filters. To test this, I’ve had people use my application and select the nodes to see if the appropriate instances are appearing. The nodes should display characteristics that coincide with the filters.

Requirements Testing

Description

“Requirements-based testing is a testing approach in which test cases, conditions and data are derived from requirements. It includes functional tests and also non-functional attributes such as performance, reliability or usability.” -https://www.tutorialspoint.com/software_testing_dictionary/requirements_based_testing.htm

Once the components are tested and the system operates as designed, the application is tested against the requirements gathered in the requirements phase of development. Ensure that all requirements map to test scenarios.

Acceptance Testing

Description

“After the system test has corrected all or most defects, the system will be delivered to the user or customer for Acceptance Testing or User Acceptance Testing (UAT). Acceptance testing is basically done by the user or customer although other stakeholders may be involved as well.” -http://istqbexamcertification.com/?s=acceptance+testing

The User Acceptance Test (UAT) validates that the system, as a whole, meets mutually agreed-upon requirements. UAT is completed by end-users of the application and occurs before a client or customer accepts the new system.
