Project Documentation Review

Application Guide

The main component for this application is the data map. When the application is loaded, the data map will be displayed. Use the legend in the bottom left to determine which colors represent with data sets. Each node on the map represents one incident. The nodes are clustered so that the map is not cluttered to the point where it is no longer legible. To view a cluster, click on it. The map with zoom in to the area that the cluster represents. You can also zoom by scrolling in/out on the mouse. Once a single node is visible on the map, you can click on it to see all of the details that correspond to that node. On the right hand side of the application are the filtering options. These affect what is visible on the map. There are two datasets that are visible in the current version of the application. The top two tabs on the right hand side of the screen split up the individual filtering options for each data set. Within these tabs are the tabs that divide the filtering options themselves. For each of these options, select the desirable filter and the map will refresh with only the nodes that fit those filters visible. The download link below the map downloads a zip file contains two csv files. These files are for each data set current available in the application. The only data that appears in these csvs is the data filtered data that is visible on the map when the download link is selected.

System Admin Guide

Since this application is hosted by shinyapps.io, there are only a few things that need to be managed. Shinyapps.io handles all of the hosting itself. The system admin would have to handle all of the user accounts and permissions associated with those accounts for people to log in to the application with. They would also need to handle the hosting tier that is currently being used for the app. If more and more people get accounts to log into the application and use it, then the shinyapps.io hosting tier needs to be upgraded.
