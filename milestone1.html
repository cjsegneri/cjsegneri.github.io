Milestone 1 - Planning Phase

Connor Segneri


Introduction

The NCMEC Data Map is a Shiny-R application that uses missing persons and attempted kidnapping data from the National Council of Missing and Exploited Children to display an interactive map with filterable characteristics for police officers to use in order to more effectively investigate and prevent kidnapping attempts.

This milestone covers the initial planning phase for the NCMEC Data Map. Every key feature and implementation process for the application is fleshed out before development begins.

Project Overview

The purpose of this project is to develop an online application that displays an interactive map that uses data collected from the National Center for Missing and Exploited Children. The goal is to provide an application to police officers across the country that allows them to visualize, filter, and download data pertaining to kidnapping and missing person information.

The National Council for Missing and Exploited Children collects data from police departments all across the country. While they have the means to store this data and collect queries from it, they do not have a user-friendly way to filter and view this data. Police officers who are working on a case and need to reference some of this data would have to request a programmer to make a specific query for them. With a more intuitive application these officers would be able to filter through the data, get what they need, solve cases, and prevent kidnapping attempts much faster. This application would ideally be accessible on secure devices in police departments across the country.

Project Scope

  • Is it a real time system?
    • This system is going to be accessible in real time. This must be the case if police are going to use it to work on current cases. This means that NCMEC is going to collect the data from police departments and feed it directly into the database that is linked to this app, so as soon as NCMEC has the data, the application can access it.
  • Is it all screen-based?
    • The application will mainly be used as a mobile desktop application that is only accessible on secure machines. A mobile application would be helpful for police officers who are currently out on patrol, but the main focus for now is to get the desktop application up and running.
  • What reports can be produced?
    • An interactive map of user-filtered data
    • A csv file containing user-filtered data
  • Where does the information come from?
    • The information is kidnapping and missing person information that comes from the National Center for Missing and Exploited Children, and they collected their data from police departments across the United States.
  • What manipulation is required for the data?
    • Various data cleaning is required
  • Normalization
  • Data type conversion
    • Appending other information to the data (like zip code data)
  • Does it store data?
    • No, it pulls data from NCMEC, they handle the storage.
  • How will the system manage security?
    • A cyber security major is going to analyze the application and provide suggestions as to what type of security is required. This also depends on what type of database NCMEC is using, as they are currently in the process of transitioning databases.

Project Assumptions

This online application might only be used by people inside of NCMEC, or it might be used by police officers across the US. The server that the application is running from must be able to handle all of the user traffic, but it is unclear what this will look like until the application is live. The data NCMEC has stored will be accessible when the application goes live. This is one of the largest assumptions that I will be making when developing this application. If the data is not there, then the application is dead. The data needs to be connected at all times, and it is being made with the assumption that it will be.
